Les concepts de base de Docker

Concepts

Image

  • Se base sur une image parente
  • Base d'un conteneur Docker
  • Liste des modifications apportées au système de fichier
  • Est décrit à l'aide d'un Dockerfile
  • class Redis { ... }

Conteneur

  • Instancie d'une image
  • Possède un état (démarée, arrêté, gelée, etc.)
  • new Redis();
  • Persistance ephémère

Concepts

Volumes

  • Permet de persister les données
  • Plusieurs moyens :
    • Docker Volume
    • Mount d'un dossier du host (mount --bind)
    • Storage Driver (Ceph, GlusterFS, etc.)

Concepts

  • Isolation des ressources
  • Image ↔ Executable + Dépendences
  • Conteneur ↔ Process + Contrôle
  • Volume ↔ Device

Concepts

Dockerfile

				FROM ubuntu:xenial
				RUN apt-get update && apt-get install -y apache2 && apt-get clean && \
					rm -rf /var/lib/apt/lists/*
				RUN apt-get update && apt-get install -y php5 libapache2-mod-php5 \
					php5-mysql php5-cli && apt-get clean && rm -rf /var/lib/apt/lists/*
				EXPOSE 80
				CMD ["/usr/sbin/apache2", "-D", "FOREGROUND"]

Concepts

Docker Engine

  • Construit une image à partir d'un Dockerfile
  • docker build -t starclay/apache-php .
  • Instancie une image comme conteneur
  • docker run -it starclay/apache-php:latest

Concepts

Docker Compose

  • Dans Dockerfile, CMD (ou ENTRYPOINT) lance un seul process
  • Une image par process (MySQL, Java, etc.)
  • Orchestration de plusieurs conteneurs
  • docker-compose up

Concepts

Docker Registry

  • Registre des images Docker disponibles
  • Docker Hub
    • https://hub.docker.com/explore/
    • Open Source
  • Possibilité de déployer un registre privé d'images Docker
  • Support du versionning des images
